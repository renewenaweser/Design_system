### 3. Drift Detector — Lean PRD
**Was es ist:** Product Requirements Document für den Open-Source GitHub Action "Design System Drift Detector".

**Beinhaltet:**

- User Stories (3 Personas: Dev, DS Lead, Head of Engineering)
- MVP Scope mit Feature-Priorisierung (MUST / SHOULD / NICE)
- Technische Architektur, `.driftrc.yml` Config-Format, GitHub Action Usage
- Go-to-Market Timeline (8 Wochen bis Open Source)

### 4. AI-Readiness Score — Deep Dive
**Was es ist:** Detaillierte Erklärung aller 8 Score-Kriterien + die echten Kundenprobleme die wir lösen.

**Beinhaltet zwei Bereiche:**

**🎯 AI-Readiness Score** — Für jedes der 8 Kriterien:
- Was es ist (einfach verständlich)
- Vorher / Nachher Vergleich
- Was WeSlam konkret beim Kunden macht (4 Schritte)

**🔥 Echte Kundenprobleme** — 6 reale Schmerzpunkte mit Business Impact in CHF:
- "Unsere AI-Lizenzen bringen nichts für UI-Code"
- "AI-generierte PRs brauchen doppelte Review-Zeit"
- "Unsere App sieht auf jeder Seite anders aus"
- u.v.m.
