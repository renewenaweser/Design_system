Was Figma als AI-Readiness definiert

🎯
Design Tokens als Fundament
Figma Report, Abschnitt: Token Infrastructure

Figma ist explizit: Tokens müssen im W3C DTCG Standard strukturiert sein. Nur maschinenlesbare Tokens ermöglichen AI die korrekte Interpretation. Der Report unterscheidet drei Ebenen:

Primitive Tokens — die Rohdaten (blue-500 = #3B82F6)
Semantic Tokens — die Bedeutung (color-primary = {blue-500})
Component Tokens — der Kontext (btn-bg = {color-primary})

Ohne diese Hierarchie greift AI auf primitive Werte zurück — und generiert hardcoded CSS statt Token-Referenzen.

🔌
Code Connect: Die Brücke zwischen Figma und Code
Figma Report, Abschnitt: Code Connect

Code Connect mappt jede Figma-Komponente auf ihre entsprechende Code-Implementierung. Ohne dieses Mapping generiert AI-Code:

  • Generic HTML (<button>) statt Design-System-Komponenten (<Button variant="primary">)
  • Inline-Styles statt Token-Referenzen
  • Eigene Implementierungen statt bestehender Bibliotheken

Mit Code Connect weiss der AI-Agent: «Diese Figma-Komponente = diese React-Komponente mit diesen Props».

🤖
Figma MCP Server: AI-Agenten lesen Figma direkt
Figma Report, Abschnitt: MCP Server

Das Model Context Protocol (MCP) ermöglicht AI-Werkzeugen wie Cursor, Claude oder GitHub Copilot, direkt in Figma-Dateien zu lesen. Das bedeutet:

  • AI versteht den Design-Intent direkt aus der Quelle
  • Kein manuelles Copy-Paste von Design-Specs
  • Design-System-Kontext fliesst automatisch in Code-Generierung ein
Figma stellt den MCP Server kostenlos zur Verfügung — aber er funktioniert nur dann optimal, wenn das Design System gut strukturiert ist. Ohne saubere Token-Hierarchie und Code Connect liefert auch MCP schlechte Ergebnisse.

Figma Prompting Framework — für WeSlam Kunden

Figma gibt im Report konkrete Prompting-Empfehlungen für AI-gestützte Design-System-Arbeit. WeSlam kann dieses Framework als Teil des Kunden-Onboardings verwenden.

Figmas 5 Best Practices für Design System Prompting

1️⃣
Komplexität aufbrechen

Grosse Design-System-Anfragen in kleine, spezifische Sub-Tasks aufteilen. Statt «Erstelle eine Produktseite» lieber «Erstelle einen Card-Component mit dem ProductCard-Token-Set».

2️⃣
Design-Kontext einschliessen

Layout-Constraints, verfügbare Komponenten, Token-Namen explizit im Prompt angeben. «Verwende die button-primary Token-Gruppe, Mindestbreite 120px, mobile-first».

3️⃣
Design System via Code Connect oder MCP verlinken

Code Connect oder MCP-Verbindung herstellen, bevor AI-Agenten genutzt werden. Ohne diese Verbindung «weiss» AI nichts über das Design System und generiert generic Code.

4️⃣
Spezifisch und plain statt vage und kreativ

«Verwende einen 16px-Button» statt «mach es grösser». «Hintergrundfarbe color-surface-primary» statt «macht es hell». Design Tokens als Sprache nutzen.

5️⃣
AI um Reasoning bitten

«Erkläre, warum du diese Komponente gewählt hast» — so erkennt man, ob AI das Design System korrekt interpretiert oder ob es einen Workaround gewählt hat.
